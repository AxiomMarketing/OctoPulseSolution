# Synergie Paid-Organic

## Quand l'utiliser
- **Hebdomadaire** : mardi matin avec Stratege (après rapports)
- Après qu'un winner clair émerge (paid OU organique)
- Avant un event saisonnier majeur (coordination messaging)

## Entrées requises
- Rapport hebdo Stratege (winners paid)
- Rapport hebdo Maeva (winners organique)
- Learnings encodés : `team-workspace/marketing/references/learnings-creatifs.md`

## Étapes

### Flux A : Paid → Organique

1. Stratege remonte un angle winner (CPA < target, MER > 3)
2. Maeva identifie le post organique équivalent à produire
3. **Reformulation obligatoire** :
   - Paid = hard hook + punch + CTA immédiat
   - Organique = storytelling + émotion + CTA soft (save, share, comment)
4. Brief via `maeva:brief-content`, mention "inspiré de [BRF paid ID]"
5. Publication dans les 7-14j (pour profiter de l'onde)

### Flux B : Organique → Paid

1. Maeva identifie un post viral organique (saves × 3, shares × 5)
2. Signal vers Stratege dans le weekly-report
3. Stratege décide GO/NO-GO pour test paid
4. Si GO : Stratege brief Forge (pas Maeva qui brief Forge) — séparation des rôles
5. Forge produit une version paid (adaptée format Meta Ads, hook punch, CTA conversion)

### Flux C : Coordination event saisonnier

1. Event à J-30 : co-construction d'un thème commun (angle + visuel + hashtag)
2. Organique monte en puissance J-21 à J-7 (awareness)
3. Paid active fort J-7 à J-1 (conversion)
4. Paid + organique overlap pendant 48h post-event
5. Debrief commun J+7

## Sortie
- Log `team-workspace/marketing/maeva/synergy/SYN-YYYY-MM-DD.md`
- SendMessage → Stratege (décision/signal)
- CC Sparky
- Encodage learning si pattern confirmé

## Règles strictes
- **Séparation des rôles** : Maeva ne brief pas Forge pour du paid. Ça passe par Stratege.
- **Voix de marque** : chaque réutilisation = reformulation. Jamais copier-coller paid → organique.
- **Attribution transparente** : si un winner vient de l'organique et bascule en paid, l'encoder comme tel dans les learnings
- **Pas de contradiction messaging** : si paid teste "angle A" et organique pousse "angle B contradictoire", alerter Sparky pour arbitrage
