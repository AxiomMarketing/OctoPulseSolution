## 2. L1 — BREAKING NEWS BYPASS LES DEFENSES ANTI-PUB

### 2.1 Origine

**Test HYP-0001 — Eruption Piton de la Fournaise (fevrier 2026)** :
- Format : Breaking News style news broadcast
- Resultat : **ROAS 23,5x**, CTR 9,58%, CPM **1,63 EUR**

Avant ce test, les creatives Univile suivaient un format "publicitaire classique" (mockup salon + texte emotionnel). CTR moyen 2-3%, CPM 8-15 EUR.

### 2.2 Le mecanisme

Le format Breaking News **mimetise l'UI editoriale** (bandeau rouge, logo "UNIVILE NEWS", ticker, "EN DIRECT") que les utilisateurs associent a du **contenu journalistique, pas publicitaire**. Resultat :

1. Le cerveau ne declenche PAS le reflexe anti-pub (banner blindness)
2. Le pattern interrupt (rouge) arrete le scroll
3. La credibilite editoriale amene l'utilisateur a lire au lieu de scroller
4. L'event evoque cree une pertinence immediate

### 2.3 Implication pratique

**Le format Breaking News / Flash Info devient le FORMAT PAR DEFAUT pour tout** :
- Lancement de nouvelle creative
- Event saisonnier (sakura, automne, Fete des Meres)
- Event reel urgent (eruption, tempete, buzz)
- Nouveau produit / nouvelle collection

### 2.4 Exemples d'application

**Evenement reel** :
> "ERUPTION HISTORIQUE — Piton de la Fournaise
> La lave a atteint l'ocean pour la 1ere fois en 19 ans."

**Evenement saisonnier** :
> "FLASH INFO — Sous les cerisiers de Kyoto, le temps s'arrete."

**Evenement cree** :
> "FLASH INFO — Automne a la Cascade Langevin
> Les pluies d'automne revelent la cascade dans toute sa force."

**Nouveau produit** :
> "EDITION SPECIALE — Mafate, capturée pour la premiere fois."

### 2.5 Nuances

- **Ne fonctionne PAS sans produit visible** (L4). Le tableau DOIT etre le heros du visuel.
- **Ne fonctionne PAS sans bandeau rouge / style news**. Un "texte dramatique" sans l'UI editoriale ne bypass pas les defenses.
- **Erosion previsible** (L7) — preparer V2 avant J3.
- **Risque de saturation** si on fait QUE du Breaking News. Garder 20-30% de creatives "classiques" (mockup, carrousel AA).

## 3. L2 — EVENT-DRIVEN DIVISE LE CPM PAR 5-10x

### 3.1 Origine

**Benchmark** :
- CPM Univile hors event : **8-15 EUR** (moyenne secteur)
- CPM Univile Eruption Piton Fournaise : **1,63 EUR**

Division par 5-10x.

### 3.2 Le mecanisme

Sur un event qui fait l'actualite :
1. L'engagement organique grimpe (meme contenu pertinent)
2. Meta detecte la pertinence et baisse le CPM (optimisation algo)
3. Les utilisateurs sont "prepared" a voir du contenu sur le sujet
4. Le CTR explose (9,58% sur Piton Fournaise) — ce qui re-abaisse le CPM

### 3.3 Implication pratique

**Fenetre de reaction : < 24h**. Un event qui a deja 48h de buzz est "fini" — les gens ont vu 500 posts dessus, l'effet pertinence se dilue.

**Priorites** :
1. Monitoring events en temps reel (Radar)
2. Capacite de production rapide (brief urgent < 2h, mode Workflow Express)
3. Breaking News template pre-configure

### 3.4 Exemples d'application

| Event | Fenetre | CPM | ROAS |
|---|---|---|---|
| Eruption Piton Fournaise (fev 2026) | 48h apres debut | 1,63 EUR | 23,5x |
| Sakura Kyoto (event cree, avril) | Saisonnier 2-4 sem | Bas | 10,19x |
| Cyclone Reunion (hypothese) | < 24h | Hypothese < 2 EUR | Hypothese 10-20x |

### 3.5 Nuances

- **Pas tous les events meritent une pub** : guerre, catastrophe humaine, sujet sensible = JAMAIS capitaliser
- **Events saisonniers** = duree de vie plus longue (2-4 semaines), erosion plus lente que events reels
- **Events CREES** (L3) fonctionnent aussi mais pas le meme niveau de CPM-bashing

## 4. L3 — FLASH INFO MARCHE SANS EVENT REEL

### 4.1 Origine

**Test HYP Sakura Kyoto** : pas d'event reel (les cerisiers fleurissent chaque annee), mais packagé en Flash Info :
- Resultat : **ROAS 10,19x**

### 4.2 Le mecanisme

Le format Breaking News / Flash Info fonctionne meme sans event reel parce que :
1. Le mimetisme editorial (L1) fait le travail — le cerveau lit en mode "news"
2. La saisonnalite naturelle (cerisiers = printemps) cree la pertinence
3. L'urgence est CREEE ("Edition limitee", "Capturé pour la premiere fois")
4. Le ton reste cinematique / poetique, pas sensationnaliste

### 4.3 Implication pratique

**On peut CREER des events** pour alimenter le format Breaking News :
- "Pluies d'automne a la Cascade Langevin"
- "Les baleines sont de retour a la Reunion"
- "Lumiere d'hiver sur le Sacre-Coeur"
- "La saison des tortues commence"

Cela garantit un flow constant de Flash Info meme quand rien ne se passe dans l'actu.

### 4.4 Exemples d'application

| Event cree | Angle | Saison | Persona |
|---|---|---|---|
| "Saison des baleines a la Reunion" | Flash Info ecologie | Juillet-Octobre | Marie, Christiane |
| "Cerisiers de Kyoto en fleur" | Flash Info poetique | Mars-Avril | Julien |
| "Levier d'or sur la Corse" | Flash Info lumiere | Oct-Nov | Julien |
| "Mafate sous la brume" | Flash Info atmosphere | Hiver austral | Marie, Christiane |

### 4.5 Nuances

- **L'event cree doit avoir un ancrage reel** (vraie saison, vrai phenomene). Pas de fake.
- **Ton = poetique/cinematique**, pas sensationnel (sinon on perd la credibilite)
- **Peut se programmer a l'avance** (vs event reel qui demande reaction rapide)

## 5. L4 — PRODUIT DOIT ETRE VISIBLE DES LA 1ERE SECONDE

### 5.1 Origine

**Tests contraires** :
- A1 "Loin de chez toi" (video brand content, produit pas visible) : **0 achat**, 231 EUR depenses, CTR 10% (piege a clics)
- A2 "Un lieu, une histoire" (carrousel narratif, produit en slide 3+) : **0 achat**, 127 EUR depenses

**Tests qui confirment** :
- B3 "Le cadeau qui touche" (produit heros du visuel) : ROAS 3,42x
- B2 Avant/Apres (produit en slide 2, mais slide 1 TEASE l'arrivee) : ROAS 69x

### 5.2 Le mecanisme

Sur Meta, l'utilisateur scroll 1-2s par post. Si le produit n'est pas identifiable en < 1s :
- Clics "curiosite" genererent du CTR mais 0 intention d'achat
- Le traffic est non qualifie
- Meta apprend une mauvaise audience

### 5.3 Implication pratique

**Regle** : **dans la premiere image / slide 1 / premiere frame, le produit (l'affiche) doit etre visible ET identifiable**.

**Exceptions acceptables** (mais rares) :
- Carrousel Avant/Apres : slide 1 = mur vide, slide 2 = produit. ACCEPTABLE car **le produit attendu est visible de loin via le mouvement de swipe**. Le slide 2 revele.

**Non acceptables** :
- Video brand content (pas de produit avant 10-15s)
- Carrousel narratif (produit en slide 3+)
- Photo paysage seul sans contexte produit

### 5.4 Exemples d'application

**CA MARCHE** :
- Statique Mockup : affiche en grand, centree
- Carrousel AA : slide 1 "mur nu" (produit attendu), slide 2 "produit revele"
- Flash Info : tableau tenu/exposé, bien visible
- Breaking News : tableau en main gantee au centre

**CA NE MARCHE PAS** :
- Video 15s de paysage avant le produit
- Photo plage sans affiche
- Texte "decouvre notre produit" sans image produit
- Illustration narrative abstraite

### 5.5 Nuances

- **L'affiche doit etre lisible** — on doit pouvoir identifier le lieu represente
- **L'affiche doit occuper minimum 25% de la surface** de l'image
- **L'affiche doit etre le point focal** (pas perdue dans le decor)

## 6. L5 — MOCKUP SALON CLASSIQUE EST MORT

### 6.1 Origine

**Pattern observe** : les creatives "Mockup salon classique" (un salon avec une affiche, style catalogue) ont :
- CTR 2,47%
- 0 achat

**Comparé a** :
- Breaking News : CTR 9,58%, ROAS 23,5x
- Carrousel Avant/Apres : ROAS 69x
- Statique Mockup **disruptif** (ex: Cascade Langevin avec eclairage dramatique) : ROAS 27x

### 6.2 Le mecanisme

Le mockup "plat" (catalogue) :
- N'arrete PAS le scroll (le cerveau est sature de ce type d'image)
- Pas de disruption visuelle
- Pas de storytelling
- Parait "generique"

### 6.3 Implication pratique

**Regle** : **un Statique Mockup doit avoir un element disruptif pour fonctionner en scroll-stopper paid**.

**Disruption possibles** :
- Lumiere dramatique (coucher de soleil dans la fenetre, ombres longues)
- Angle inhabituel (vue du sol, contre-plongee, vue plongeante)
- Contexte narratif (personne lisant sur le canape, tasse fumante, chien)
- Composition forte (affiche enorme, panoramique, hors proportion)
- Couleur dominante (piece entierement terracotta, vert sauge)

### 6.4 Exemples d'application

**Mockup classique MORT** :
- Salon beige, mur blanc, affiche centree, pas de lumiere particuliere, pas d'ambiance
- = catalogue IKEA

**Mockup disruptif VIVANT** :
- Salon chaleureux au coucher du soleil, affiche eclairee dramatiquement
- Vue depuis le canape, affiche qui domine le champ de vision
- Personne avec cafe fumant, affiche en arriere-plan

### 6.5 Nuances

- **Le mockup classique n'est pas MORT en organique/email** : il marche tres bien pour creer un feed coherent Instagram ou un email Welcome chaleureux
- **Il est MORT uniquement en paid scroll-stopper**
- **Il peut rester en slide 2-3 d'un carrousel** (après le hook disruptif du slide 1)

## 7. L6 — PRIX DOIT ETRE MENTIONNE

### 7.1 Origine

**Tests contraires** :
- A1 "Loin de chez toi" (pas de prix) : CTR 10%, 0 achat (clics curiosite)
- A2 "Un lieu, une histoire" (pas de prix) : 0 achat

**Tests qui confirment** :
- B3 "Le cadeau qui touche" (prix mentionne) : ROAS 3,42x
- B2 Avant/Apres (prix mentionne) : ROAS 69x
- Breaking News Eruption (prix mentionne) : ROAS 23,5x

### 7.2 Le mecanisme

Le prix affiche :
1. **Qualifie le trafic** — seuls ceux que le prix ne rebute pas cliquent
2. **Evite les clics curiosite** qui ne convertiront jamais
3. **Ancrage cognitif** — "des 28 EUR" positionne Univile comme accessible
4. **Reassurance** — transparence = confiance

**Sans prix** :
- Clics de curiosite ("combien ca coute ?")
- Decouverte du prix sur la page = bounce souvent
- Meta apprend une mauvaise audience (clicker sans intention d'achat)

### 7.3 Implication pratique

**Toujours mentionner au moins une de ces formules** :
- "A partir de 28 EUR" (prix d'appel)
- "Livraison offerte des 70 EUR" (push panier moyen)
- "Des 28 EUR, livraison offerte des 70 EUR" (combinaison ideale)

### 7.4 Placement dans le texte

**Meilleur placement** :
- Primary text, derniere ligne : "A partir de 28 EUR. Livraison offerte des 70 EUR."
- Dans le visuel (overlay en bas ou coin) : "DES 28 EUR"
- Sur le slide 3 (carrousel) : close-up produit + prix + CTA

**Mauvais placement** :
- Au milieu d'une phrase ("notre affiche a 28 EUR de la Reunion")
- Caché dans une ligne longue
- Sur un fond qui ne contraste pas

### 7.5 Nuances

- **En organique** : le prix est souvent absent (CTA "lien en bio"). C'est OK car contexte different.
- **En email** : le prix peut etre hors image (dans le HTML Klaviyo).
- **Pas de prix affiche dans Stories** si le produit renvoie directement a la page produit avec prix clair.

## 8. L7 — EROSION BREAKING NEWS EST PREVISIBLE

### 8.1 Origine

**Pattern observe** sur Breaking News (Eruption Piton Fournaise) :
- J1 : ROAS 32x
- J2 : ROAS 23x
- J3 : ROAS 7x
- J4+ : ROAS < 5x

Le ROAS **divise par 4-5 en 3 jours**.

### 8.2 Le mecanisme

Le format Breaking News fonctionne sur la pertinence temporelle :
1. J1-J2 : l'event est au pic de l'actualite, ultra-pertinence
2. J3 : la pertinence commence a decliner (les gens ont deja vu, pensée a autre chose)
3. J4+ : le format devient "du vieux" — perte du mimetisme editorial (plus "news")

### 8.3 Implication pratique

**Regle** : **preparer la V2 AVANT le J3**, idealement dans les 24h suivant le lancement.

**Types de V2** :
- V2 "La suite" : "La lave a cesse", "Les dernieres images"
- V2 "Retour au calme" : changement d'angle emotionnel
- V2 "Souvenir" : "Il y a 1 semaine..."
- V2 Nouvelle angle : passer de BREAKING NEWS a EDITION SPECIALE

### 8.4 Workflow d'anticipation

| Timing | Action |
|---|---|
| **J1** (lancement V1) | Produire V2 en parallele (meme brief, angle different) |
| **J2** | V2 prete et validee par Marty |
| **J3** (erosion V1) | Lancer V2, kill V1 progressivement |
| **J5** | Preparer V3 si event encore pertinent |
| **J7** | Fin du cycle — retour aux formats evergreen (Carrousel AA, Mockup) |

### 8.5 Exemples d'application

**Event reel — Eruption Piton Fournaise** :
- V1 J1-J3 : "ERUPTION HISTORIQUE — La lave atteint l'ocean"
- V2 J3-J5 : "EDITION SPECIALE — La lave a cesse"
- V3 J5-J7 : "DERNIERES IMAGES — Avant que le volcan se rendorme"
- Evergreen J7+ : Carrousel Avant/Apres avec l'affiche Coulee 2026

**Event saisonnier — Sakura** :
- Duree de vie plus longue : 2-4 semaines
- V1 J1-J7 : "FLASH INFO — Les cerisiers sont en fleurs"
- V2 J7-J14 : "Le pic de floraison commence"
- V3 J14-J21 : "Dernieres semaines pour capturer l'instant"

### 8.6 Nuances

- **L'erosion n'est pas la meme pour events reels vs saisonniers**
- **Events saisonniers** : erosion sur 2-4 semaines (vs 3-4 jours pour events reels)
- **La V2 doit etre coherente** avec la V1 (meme univers, meme produit)

## 9. L8 — FUNNEL N'EST PAS LE PROBLEME

### 9.1 Origine

**Donnees funnel Univile (30 achats, feb-mars 2026)** :
- View Content → Add to Cart : **11,63%**
- Add to Cart → Purchase : **91,38%**

**Benchmarks industrie** :
- VC → ATC : 3-8% (Univile superieur)
- ATC → Purchase : 30-60% (Univile superieur)

### 9.2 Le constat

Le funnel est **excellent**. Univile convertit bien une fois que les gens arrivent sur la page produit et ajoutent au panier.

**Donc** : les problemes de performance paid ne viennent PAS :
- De la page produit (elle convertit bien)
- Du process de checkout (91% convertissent apres ATC)

**Ils viennent** des CREATIVES / du TRAFIC :
- Creatives qui n'attirent pas le bon public
- Creatives qui genererent du trafic non qualifie (clics curiosite)
- Creatives qui ne convertissent pas suffisamment d'attention en clics

### 9.3 Implication pratique

**Regle** : **si une creative underperform, le probleme est SUR la creative, pas sur le funnel downstream**.

Il ne faut PAS :
- Rebuilder la page produit
- Changer le checkout
- Ajouter des pop-ups de retention

Il FAUT :
- Iterer la creative
- Tester d'autres angles
- Verifier la coherence creative ↔ page produit (ad-to-page match)

### 9.4 Exemples d'application

**Creative underperform** → **diagnostic** :

| Symptome | Probable cause | Action |
|---|---|---|
| CTR bas (< 1%) | Scroll-stopper faible | Changer le visuel (disruption, Breaking News) |
| CTR haut mais ATC bas | Trafic non qualifie (prix pas mentionne, angle piege) | Ajouter prix, reformuler CTA |
| ATC OK mais achats bas | Rare (funnel ATC→Purchase = 91%) | Verifier si bug checkout, sinon = outlier |

### 9.5 Nuances

- **Nexus surveille la conversion on-site** et envoie des insights a Forge
- **Si Nexus detecte un probleme on-site** (ex: nouvelle page qui convertit mal), on corrige on-site
- **Mais en regime de croisiere, le funnel est stable** et le point d'amelioration = creatives

## 10. L9 — AD SET B PORTE 100% DES VENTES

### 10.1 Origine

**Donnees par Ad Set (30 achats)** :
- Ad Set A (Broad) : 247 EUR depenses, **0 achat**
- Ad Set B (Interets / LAL) : depense moyenne, **30 achats**
- Ad Set C (Test) : budgets tests variables, **0 achats sur les angles echoues, 3+ ATC sur les winners**

### 10.2 Le piege interpretatif

**Interpretation fausse** : "Broad ne marche pas, abandonner Broad."

**Interpretation correcte** : "Les creatives envoyees a Broad n'etaient pas adaptees au Broad. Broad peut marcher avec les bonnes creatives."

### 10.3 Pourquoi Broad a echoue

Les creatives testees en Broad :
- Etaient ciblees persona Marie (Diaspora) avec langage intime "ton ile"
- Broad = audience tres large = tous les CSP, tous les ages, toutes les origines
- Un message "ton ile" sur Broad = 95% d'audience n'ayant pas d'ile

**Pour qu'une creative marche en Broad**, il faut :
- Angle universel (Transformation, Cadeau generique)
- Demo visuelle forte (Avant/Apres, Flash Info event)
- Texte plus universel (moins "diaspora")

### 10.4 Implication pratique

**Regle** : **concevoir certaines creatives SPECIFIQUEMENT pour le Broad**.

**Creatives Broad-ready** :
- Carrousel Avant/Apres sur lieu France (Paris, Corse) — universel
- Flash Info evenement (sakura, eruption — tout le monde reagit)
- Statique Mockup cadeau "universel" (pas diaspora)

**Creatives Interets/LAL (garder ciblage precis)** :
- Angle diaspora Marie
- Fierte reunionnaise Christiane
- Niche design Julien

### 10.5 Exemples d'application

| Creative | Cible ideale | Raison |
|---|---|---|
| B2 Avant/Apres Reunion | LAL Diaspora | Langage "ton ile" |
| **Julien Paris Avant/Apres** (a tester) | Broad + Interets Design | Universel |
| Sakura Flash Info | Broad + Interets Japon/voyage | Event universel |
| Christiane Cadeau Reunion | LAL Reunion 55+ | Persona tres specifique |

### 10.6 Nuances

- **Ne PAS abandonner Broad** — il faut lui donner les BONNES creatives
- **Broad = faible CPM** mais faible intention — besoin de creatives tres fortes en scroll-stop
- **LAL/Interets = intention plus forte** mais CPM plus eleve

## 11. L10 — CREATIVES EN FAMINE N'ONT PAS ETE TESTEES

### 11.1 Origine

**Observation** : sur 30 achats, certains personas (Julien notamment) ont **0 achat**. Conclusion tentante : "Julien ne convertit pas."

**Mais** : en regardant les budgets, tous les tests Julien ont eu **< 10 EUR depenses sur 7 jours**. 

En Meta Ads, avec un budget aussi faible :
- L'algo Meta ne recoit pas assez de signaux
- Il "ignore" la creative (pas assez de donnees pour optimiser)
- Il diffuse a une audience random, pas aux plus interesses
- Resultat : CTR bas, ATC bas, 0 achat — mais **pas parce que la creative est mauvaise**, parce qu'elle a ete **etouffee**

### 11.2 Le mecanisme

Budget minimum Meta pour "reel" test :
- **50 EUR minimum** sur 5-7 jours
- Ou **2x CPA target** (donc 56 EUR si CPA cible = 28 EUR)

**Sous ces seuils** : famine = non concluant.

### 11.3 Implication pratique

**Regle** : **ne JAMAIS conclure qu'une creative / angle / persona "ne marche pas" si le budget test etait < 50 EUR**.

**Consequence majeure pour Julien** : il n'a jamais ete teste correctement. Il doit etre **re-teste en 2026 avec budget plein (50-100 EUR minimum sur 7j)** avant de conclure quoi que ce soit.

### 11.4 Kill rule vs famine

| Situation | Verdict |
|---|---|
| Depense 50+ EUR, 0 achat, ROAS < 1,55x (breakeven), CPA > 1,8x cible | **KILL** — creative ou audience mauvaise |
| Depense < 50 EUR, 0 achat | **FAMINE** — relancer avec budget suffisant |
| Depense 50+ EUR, 1-2 achats, ROAS entre breakeven et 3x | **OPTIMISER** — pas kill, iterer |
| Depense 50+ EUR, ROAS > 3x | **SCALER** — +20%/j |

### 11.5 Exemples d'application

**Julien persona (hypothese)** :
- Historique : 0 achat, mais budgets tous en famine
- Plan 2026 : Budget 100 EUR sur 7j, format champion (Carrousel AA), lieux France
- **Seulement apres ce test**, on pourra conclure si Julien "convertit" ou non

**Christiane persona** :
- Jamais testee
- Meme principe : test budget plein (50-100 EUR/7j) requis pour conclusion valide

### 11.6 Nuances

- **La famine n'est pas toujours volontaire** — Meta peut "famine" une creative malgre un budget correct si le CPM monte anormalement
- **Verifier la depense REELLE**, pas le budget alloue
- **Les tests de "confirmation" de winners** doivent aussi avoir un budget plein

## REGLES TRANSVERSES (rappel)

1. **Les donnees pilotent** — jamais l'intuition. Chiffres > hypotheses.

3. **L'historique est la source de verite** — consulter AVANT chaque production.
4. **Le feedback loop est obligatoire** — apres chaque test, encoder le learning.
