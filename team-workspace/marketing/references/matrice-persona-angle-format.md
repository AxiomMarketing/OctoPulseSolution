## 1. METHODOLOGIE DE LA MATRICE

Chaque **brief creatif = 1 persona + 1 angle + 1 format + 1 lieu**. La matrice permet de :

1. **Eviter les doublons** : si une combinaison a deja echoue (ROAS 0), on ne la re-teste pas
2. **Identifier les gaps** : combinaisons jamais testees = opportunites
3. **Prioriser les tests** : combinaisons avec forte probabilite de succes (format champion + persona prouve sur autre angle) passent en priorite

### Colonnes de la matrice

| Colonne | Signification |
|---|---|
| **Persona** | Marie / Julien / Christiane |
| **Angle** | Transformation / Cadeau / Nostalgie / Evenement / Saisonnier / Collection |
| **Format** | Carrousel AA / Statique Mockup / Flash Info / Carrousel Multi / autres |
| **Exemple / lieu** | Creative specifique ou lieu concret |
| **ROAS** | Performance mesuree (si testee) |
| **CPA** | Cout par achat |
| **Statut** | TESTE / A TESTER / INTERDIT |
| **Predictions** | Hypothese de perf si jamais teste |

### Sources de verite

- **Historique** : `Workspace/historique-creatif.md`
- **Learnings** : section 1.7 du SOUL Forge
- **Feedback Stratege** : apres chaque cycle de test
